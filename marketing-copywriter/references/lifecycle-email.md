## Email Method

1. Identify the subscriber or contact trigger.
2. Define the next useful action, not only the business goal.
3. Choose the lane: education, proof, product, activation, retention, reactivation.
4. Write one main idea per email.
5. Tie the CTA to the reader's current intent and trust level.
6. Keep the sequence varied so it does not feel like repeated asks.
7. Review for deliverability, compliance, and expectation-setting.

Best practices:

- Deliver the thing promised immediately.
- Respect the inbox as a relationship channel.
- Use plain language and concrete value.
- Let each email earn the next open.
- Keep a consistent sender voice.

Anti-patterns:

- Asking for a demo before showing relevance.
- Sending three emails in a row with the same sales ask.
- Subject line that creates curiosity the email does not satisfy.
- Newsletter with five unrelated ideas and no thesis.
- "Checking in" with no value.

## Sequence Strategy

Define:

- audience
- trigger
- current belief
- desired belief
- desired action
- objections to resolve
- proof to introduce
- offer or conversion point

Sequence types:

- **Welcome**: orient and deliver value.
- **Trial onboarding**: drive activation and reduce setup friction.
- **Demo nurture**: build belief before or after sales contact.
- **Lead magnet follow-up**: convert resource interest into problem/solution awareness.
- **Customer onboarding**: reduce churn and guide first value.
- **Reactivation**: re-open a dormant relationship with a useful reason.
- **Newsletter**: create ongoing trust and memory.

## Onboarding Arc

Default arc:

1. **Deliver the promise**: give the resource/access and set expectations.
2. **Quick win**: one action they can complete now.
3. **Belief shift**: story, metric, or example that changes how they see the problem.
4. **Depth**: guide, product education, segment-specific path.
5. **Proof**: customer outcome, quote, before/after.
6. **Soft conversion**: reply, demo, setup, trial step, checklist, consultation.

Timing principle:

- Early emails should reduce friction.
- Middle emails should build belief.
- Later emails can ask for more commitment.

## Nurture Lanes

Educational lane:

- Job: teach something specific.
- CTA: read, watch, use, check, compare.
- Risk: over-teaching without connecting to the offer.

Proof lane:

- Job: make the claim believable.
- CTA: see case, view example, book call, compare result.
- Risk: vague customer story with no metric or tension.

Product lane:

- Job: connect feature to outcome.
- CTA: try, configure, compare, talk to sales.
- Risk: feature tour without reader problem.

Activation lane:

- Job: move user to first value.
- CTA: complete setup, invite teammate, import data, run report.
- Risk: too many steps at once.

Retention lane:

- Job: deepen value after purchase.
- CTA: adopt feature, review workflow, train team, expand use case.
- Risk: sounding like acquisition copy to existing customers.

## Message Structure

Useful single-email shape:

```markdown
Subject:
Preview:
Opening hook:
Reader context:
One useful idea:
Proof or example:
CTA:
P.S.:
```

Opening hook options:

- name the moment
- challenge a false assumption
- state a useful observation
- give a specific outcome
- acknowledge a common friction

Body rules:

- Put the main point above the fold.
- Keep paragraphs short.
- Avoid over-explaining what the reader already knows.
- Use one CTA unless there is a deliberate primary/secondary path.
- Make links descriptive.

## Subject Lines

Subject line jobs:

- signal relevance
- create honest curiosity
- promise a clear benefit
- remind the reader of a trigger or task

Good patterns:

- `How [role] handles [specific problem]`
- `[Template/resource] for [task]`
- `A faster way to [outcome]`
- `Before you [action], check [risk]`
- `[Customer] cut [metric] without [pain]`

Avoid:

- false urgency
- bait-and-switch curiosity
- fake personalization
- vague "quick question"
- all-caps pressure
- "You won't believe..."

Preview text:

- Do not waste it repeating the subject.
- Use it to add context, proof, or the next detail.

## CTA and Segmentation

CTA by stage:

- Awareness: read guide, use checklist, compare approaches.
- Consideration: view case study, watch demo, calculate impact.
- Decision: book call, start trial, see pricing, request migration plan.
- Customer: complete setup, invite team, enable feature, schedule review.

Segmentation signals:

- role
- industry
- company size
- source asset
- product usage
- lifecycle stage
- declared interest
- sales status

If segmentation is unknown, avoid overly specific assumptions.

## Review Rubric

Score 0-2:

- Trigger clarity
- One main idea
- Reader value
- Proof or example
- CTA fit
- Sequence variety
- Voice consistency
- Compliance risk

## Common Findings

Value gap:

- Evidence: email exists only to ask.
- Fix: add a useful idea, proof, resource, or specific next step.

Sequence repetition:

- Evidence: multiple emails use same argument and CTA.
- Fix: rotate education, proof, and product lanes.

Subject mismatch:

- Evidence: subject promises one thing, body delivers another.
- Fix: rewrite subject after final body draft.

Premature ask:

- Evidence: high-friction CTA before trust is established.
- Fix: add proof or lower-friction CTA.

Generic newsletter:

- Evidence: multiple disconnected items with no thesis.
- Fix: lead with one point of view and make links support it.
